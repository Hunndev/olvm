# OLVM 사이트 등록 인터뷰

## 호출 타이밍
- `/olvm-survey {company} {site}` 호출 시 사이트 정보가 없거나 미완성 상태인 경우
- 정기 점검 시 변경 사항 확인 흐름

## 핵심 원칙

### 1. 한 번에 한 항목씩
사용자에게 한 번에 하나만 질문. 답변을 받기 전에 다음 항목으로 넘어가지 말 것. 사용자가 모른다고 하면 `confirm-commands.md` 의 확인 명령어를 안내하고, 그래도 모르면 "(미확인)" 처리하고 다음.

### 2. 답변마다 베이스라인 대비 검증
`baseline.md` 를 참조해 답변이 정상 범위인지 검증. 이상한 답이면 다시 묻거나 우려 사항으로 기록.

예시:
- "Engine 형태가 standalone인데 호스트가 1대" → 정상
- "Engine 형태가 SHE인데 호스트가 1대" → 비정상. SHE는 최소 2대 권장. 재확인.
- "스토리지망이 관리망과 같음" → 운영 위험. 알려진 이슈로 기록 권장.

### 3. 즉시 저장
항목 하나 완료될 때마다 사이트 `.md` 파일을 갱신. 중간에 세션 끊겨도 손실 없음.
- frontmatter 필드는 즉시 갱신
- 본문 섹션은 항목별로 채움
- 누락된 필드는 `pending_fields` 배열에 기록

### 4. 사용자 페이스 존중
- "한꺼번에 표로 붙여넣을게요" → 받아서 파싱
- "한 항목씩 가요" → 그렇게 진행
- "지금 시간 없어요, 나머지 다음에" → 진행 상태 저장하고 종료

### 5. 사이트 디렉토리 자동 생성
- `~/.olvm/sites/{company}/` 디렉토리 없으면 먼저 만들기
- `_company.md` 없으면 회사 등록 인터뷰부터 진행 (회사 공통 정보)
- 그 다음 사이트 인터뷰

## 9단계 인터뷰 흐름

상세 항목은 `checklist.md` 참조. 큰 흐름:

1. **기본 정보** — 위치, 운영 시간, 점검 윈도우, 현장 출입 절차
2. **OLVM 환경** — 버전, OL 버전, Engine 형태 (standalone / SHE), 콘솔 URL
3. **호스트 인벤토리** — 호스트별 IP, iLO, 역할, CPU, RAM, 디스크
4. **네트워크** — 관리망/VM망/스토리지망/마이그레이션망 VLAN/대역/본딩
5. **스토리지** — 종류, 마운트, 도메인, SPM 호스트
6. **인증/접속** — 콘솔 URL, LDAP/AD 연동, Bastion
7. **백업** — 명령, 위치, 보존, 마지막 복구 테스트
8. **모니터링** — 솔루션, 채널, 임계치
9. **알려진 이슈 / 변경 시 주의사항**

## 단계 1 진입 전 — 회사 컨텍스트 확인

```
1) ls ~/.olvm/sites/ 로 회사 디렉토리 목록 확인.
2) 사용자에게 어느 회사인지 묻기. {company} 인자로 주어졌으면 그대로 사용.
3) ~/.olvm/sites/{company}/_company.md 존재 여부 확인:
   - 없음: "이 회사는 처음 등록입니다. 회사 공통 정보부터 채울게요."
           → _TEMPLATE/_company.md 복사 → 회사 인터뷰 진행 → 저장
   - 있음: 회사 공통 정보 로드. 사이트 인벤토리 표에 이 사이트 추가 예정 표시.
4) 사이트 인터뷰 진행. _TEMPLATE/site.md 복사 → ~/.olvm/sites/{company}/{site}.md 생성.
5) 9단계 항목별로 묻고 답받고 저장.
```

## 저장 규칙

- frontmatter 필드 갱신은 명확한 답변 시에만. "(미확인)"은 본문에만 표시하고 frontmatter는 빈 값 유지.
- `pending_fields` 배열에 누락 필드 키 누적. 다음 호출 시 이 배열로 미입력 항목 식별.
- `last_updated` 매 저장마다 갱신 (ISO 날짜).
- 사용자가 명시적으로 "끝" 또는 "중단" 하면 현재 상태로 저장 후 종료.

## 인터뷰 종료 시 안내

```
등록 완료. 채워진 항목 N개 / 미확인 M개.
미확인: [field1, field2, ...]

나중에 채우는 방법:
- 현장 가서 확인할 거면: /olvm-survey 호출 → "현장 가서 채울게요" 답변 → 인쇄 모드
- 사무실에서 이어서: 같은 명령어 호출 → 미입력 항목부터 이어서 진행
```
